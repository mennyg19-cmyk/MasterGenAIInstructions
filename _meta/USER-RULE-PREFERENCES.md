# Menny's Rule Preferences (canonical)

Resolved 2026-06-10. Ponytail integrated always-on; these choices win when ponytail and MasterGenAI Instructions disagree. Copied into each project's README § Rule Preferences via template.

| Area | Choice |
|---|---|
| **Unknown conflict** | Default protocol-safe, keep building, tell user + offer to record in README § Rule Preferences |
| **Ponytail intensity** | **full** (always on every session) |
| **Rebuild scope** | Ask when a feature feels speculative/bolted-on; don't auto-drop |
| **Dependencies** | Ponytail ladder — no new package unless stdlib + native + existing deps fail |
| **Testing** | Hybrid — testing-protocol floor, ponytail-shaped minimal tests |
| **Walkthrough headers** | **Off** (archived; `enable walkthroughs` to restore) |
| **Chat style** | Split — terse for routine edits; full prose when user asks "explain" or for decisions/conflicts |
| **Reviews** | Hybrid + **strong ponytail-review** mandatory at every phase gate |
| **Complex requests** | Fix-don't-suggest — build what was asked, no re-arguing |
| **God files** | Split on refactor or god file (>500 lines / mixed concerns) — token-aware |
| **Multi-model subagents** | Full protocol for rebuild/redesign only; single-model elsewhere unless user says "use more models" |
| **Verification** | Tiered — smoke for small fixes; full checklist + running app for features/rebuilds/phases |
| **Protocol artifacts** | Ask when an artifact feels heavy; otherwise use protocol formats |
